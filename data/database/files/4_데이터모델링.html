  1. 데이터모델링 정의

    2.     : 현실 세계에 존재하는 데이터를 컴퓨터 세계의 데이터베이스로 옮기는 변환 과정.

  2. 데이터모델링 내부

    1. 개념적 모델링
      2. : 현실 세계에 존재하는 데이터에 대한 중요 데이터를 추출하여 개념 세계로 옮기는 작업

    2. 논리적 모델링
      3. : 개념 세계의 데이터를 데이터베이스에 저장할 구조를 결정하고 이 구조로 표현하는 작업.

  3. 데이터 모델

    1. 데이터 모델 정의

      2.  : 데이터 모델링을 쉽게 할 수 있도록 도와주는 도구

        - ) 개념적 데이터 모델 : 사람의 머리로 이해할 수 있도록 현실 세계를 개념적 모델링하여 데이터베이스의 개념적 구조로 표현하는 도구.
        - ) 논리적 데이터 모델 : 개념적 구조를 논리적 모델링하여 데이터베이스의 논리적 구조로 표현하는 도구.

    2. 데이터 모델 구성

      3. 1 . 데이터 구조
      2 . 데이터 연산
      3 . 데이터 제약조건

  4. 개체-관계 모델

    5.   : 개체(entity)와 개체 간의 관계(relationship)를 이용해 현실 세계를 개념적 구조로 표현하는 방법.

    1. 개체
      2. : 개체는 현실 세계에서 조직을 운영하는 데 꼭 필요한 사람이나 사물과 같이 구별되는 모든 것을 의미.
    2. 속성
      3. : 속성(attribute)은 개체가 가지고 있는 고유의 특성.
    3. 관계
      4. : 관계(relationship)는 개체와 개체가 맺고 있는 의미 있는 연관성으로 , 개체-관계 모델의 중요한 요소.
