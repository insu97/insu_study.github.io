  1. 관계 데이터 모델의 기본 용어

    2. - 릴레이션 : 관계 데이터 모델에서 하나의 table역할.

    1. 속성 : 릴레이션의 열을 속성 또는 attribute라고 한다.
    2. 투플 : 릴레이션의 행을 tuple이라고 한다.
    3. 도메인 : 속성 하나가 가질 수 있는 모든 값의 집합을 해당 속성의 domain이라고 한다.
    4. NULL 값 : 릴레이션에 있는 특정 투플의 속성 값을 모르거나 , 적합한 값이 없는 경우에는 null이라는 특별한 값 사용
    5. 차수 : 하나의 릴레이션에서 속성의 전체 개수를 차수(degree)라고 한다.
    6. 카디널리티 : 하나의 릴레이션에서 투플의 전체 개수를 릴레이션의 카디널리티라고 한다.

  2. 릴레이션의 특성

    1. tuple의 유일성 : 하나의 릴레이션에는 동일한 tuple이 존재할 수 없다.
    2. tuple의 무순서 : 하나의 릴레이션에서 tuple 사이의 순서는 무의미하다.
    3. 속성의 무순서 : 하나의 릴레이션에서 속성 사이의 순서는 무의미하다.
    4. 속성의 원자성 : 속성 값으로 원자 값만 사용할 수 있다.

  3. 키의 종류

    1. 슈퍼키 : 유일성의 특성을 만족하는 속성 또는 속성들의 집합.
    2. 후보키 : 유일성과 최소성을 만족하는 속성 또는 속성들의 집합.
    3. 기본키 : 데이터베이스 설계자나 관리자는 여러 후보키 중에서 기본적으로 사용할 키를 반드시 선택해야 하는데 이것이 기본키다.

      4. - . null값을 가질 수 있는 속성이 포함된 후보키는 기본키로 부적합하다.
      - . 값이 자주 변경될 수 있는 속성이 포함된 후보키는 기본키로 부적합하다.
      - . 단순한 후보키를 기본키로 선택한다.

    4. 대체키 : 기본키로 선택되지 못한 후보키들.
    5. 외래키 : 어떤 릴레이션에 소속된 속성 또는 속성 집합이 다른 릴레이션의 기본키가 되는 키.
